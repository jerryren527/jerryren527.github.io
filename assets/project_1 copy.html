Employee/HR Website Portal

Mar 3rd 2023 10 Min read

Goal

Create a employee/HR website portal for managing the employee onboarding process. Employees will be able to update personal information, upload required documents for identification and work authorization, and monitor their housing situation. HR will be able to access employee information and make changes regarding the onboarding status or housing assignments.

Features

For Employees

  • Registration Page
    • A new employee can access the registration page from a registration token and link that was generated by an HR representative and sent via email.
    • Users must provide unique username, password, and email.
  • Login Page
    • Users provide their username and password to log in, after which they are redirected to the Personal Information page.
    • If the user has valid token, then after refreshing the page, they should still be logged in
  • Onboarding Application page
    • Onboarding applications can have one of three statuses: Pending, Approved, Rejected.
    • A user should be redirected to this page after they log in if they have never submitted an application, or if their application status if Pending or Rejected.
    • If approved, a user should be redirected to the home page.
  • Personal Information Page
    • Name, Address, Contact Info, Visa Title, Emergency Contact, Documents
  • Visa Status Management Page
    • Users with OPT visas should be able to manage their work authorization documents. This page is used to track the status of their uploaded documents and required next steps.
    • Documents should be uploaded in this order, one by one. Users can only upload the next document after the previous document has been approved by HR
      1. OPT Receipt (submitted in onboarding application, waiting for approval)
        1. If pending, there is a message: “Waiting for HR to approve your OPT Receipt”.
        2. If approved, there is a message “Please upload a copy of your OPT EAD”.
        3. If rejected, users see HR’s feedback
      2. OPT EAD
        1. If pending, there is a message: “Waiting for HR to approve your OPT EAD”.
        2. If approved, there is a message “Please download and fill out the I-983 form”.
        3. If rejected, users see HR’s feedback
      3. I-983
        1. Users are able to download two documents, "Empty Template" and "Sample Template"
        2. If pending, there is a message: “Waiting for HR to approve and sign your I-983”.
        3. If approved, there is a message “Please send the I-983 along with all necessary documents to your school and upload the new I-20”.
        4. If rejected, users see HR’s feedback
      4. I-20
        1. If pending, there is a message: “Waiting for HR to approve your I-20”.
        2. If approved, there is a message “All documents have been approved”.
        3. If rejected, users see HR’s feedback
  • Housing Page
    1. House Details (address, list of roommates)
    2. Facility Reports
      • Creating a Facility Report
      • Viewing existing reports
      • Adding or updating comments on a Facility Report
      • Viewing a report's list of comments

For HR

  • Login Page
    • This is exactly the same as the employee’s login page. The system detects whether an account is for an employee or HR.
  • Employee Profiles Page
    • Allows HR to see a summary of each employee’s profile, search for a particular employee, and view their entire profile.
  • Visa Status Management Page
    1. In Progress: Lists all employees who have not yet uploaded and been approved for all required OPT documents.
      1. For each employee:
        1. Name
        2. Work Authorization
        3. Next Steps
        4. Approve/Reject and Feedback
    2. All: List all all employees. This is for looking at their uploaded documents. HR can search and filter for a particular employee.
  • Hiring Management Page
    • HR can generate registration tokens and review onboarding applications. HR can generate a token and send an email with a link. Clicking on the link takes them to the registration page.
    • There is a listed history of email addresses that received a registration link. It shows the email address, person’s name, registration link, and status for whether this email address has been used to successfully register an account
    • Onboarding Application Review
      1. Pending: HR can view and approve/rejects all submitted, pending onboarding applications.
      2. Rejected: HR can view all submitted, rejected applications.
      3. Approved: HR can view all submitted, approved applications.
  • Housing Management Page
    • HR should be able to view existing houses, add new houses, and delete houses.
    • Summary View
      • Address, landlord, number of employee residents
      • Facility reports and comments
      • Employee residents information
    • Adding Houses by providing an address, landlord information, basic facility information
